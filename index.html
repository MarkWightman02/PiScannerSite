About

The Raspberry Pi RFID Scanner is a project that seeks to create a new system for taking attendance in a school or workplace enviornment

  • Set-up and Go! After the initial setup, the scanner requires little user interaction!
  • Low Overhead! This system is budget friendly to implement!
  • Simplify Your Attendance! By removing a user, the Raspberry pi RFID Scanner will make taking attendance much simpler!

Implementation

Implementation of the PI Scanner system will vary depending on the enviornment and any customizations made. However, the high-level steps have been provided below.

  • Set-up an admin scanner in a secure location where only the IT department can reach it
  • Install at least one user scanner into a location where the scanner portion is accessible. Some options for this include installing the case into the wall, leaving the scanning side facing outwards or installing the board behind the wall while keeping the scanner on the outside of the wall.
  • Power both scanners. This is done by using a micro-usb cable from a standard wall outlet into the board.
  • Create ID cards for all required members of the organization.
  • Distribute ID cards to members

Dev/Design Approach

Design Process

The design process for the project was an iterative process including research, diagrams, prototyping and design.

Research

Going into the project, I had very little exposure to the technology I would be using. I had to learn about GPIO pins, Python, PHP, and HTML languages as well as how to use project management tools such as Git and Trello.

Diagrams

Instead of going straight from research to design, I created diagrams such as the ones below in order to keep myself organized. Diagrams helped to simplify the design process because I had everything that I would designing laid out.

Local environment: The user will swipe their RFID card on the scanner, after that, the user is finished with their interaction. The scanner will then send the RFID of the card and the time it was swiped to a remote database.

Remote environmental: The remote database houses a table which pairs RFID numbers and names, and a table which pairs names with times that the card was swiped. When the database receives an RFID and time, it makes a new row in the table and displays that someone swiped their card at some time. That information is displayed on the website in a user-friendly format.

Creation of RFID cards: An IT administrator or equivalent will swipe an RFID card on an admin scanner. If the RFID is not registered with a name, the scanner will prompt for a name to be registered. That name will then be updated in the database to be attached to the RFID. If the RFID is registered with a name, the scanner will prompt that the name will be overwritten. A new name will then be entered and it will update in the database.

Prototyping

I made prototypes for each part of the project instead of going straight to designing. Prototypes helped me work out errors that would have been problematic if I had started designing without catchign them. During this stage, I used breadboards to wire modules into the Pi's due to the ease of access to the pins. I also used Git in order to prototype the website and code in order to make sure I always had a folder with code that I know worked.

Design

Designing the project was where I learned the most. Through trial and error, I learned how GPIO pins work and how I can manipulate computer languages to my liking. After the modules were wired to the pi without the breadboard, I integrated the code. The logic behind the Pi scanner is written in Python, I had to manipulate this code to send data to a remote database which was used for the website. The website was written with HTML, CSS, and PHP.
